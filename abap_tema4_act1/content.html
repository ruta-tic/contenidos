    Título y tipo en la creación de un programa ABAP

    Título

    Debe ser lo más descriptivo posible. En este caso será “Mi primer programa ABAP”.

    Tipo

    Se refiere al tipo de objeto que se está creando. Generalmente, será “Programa ejecutable”.

    1. Presione grabar y el sistema, en una nueva ventana, le solicitará que ingrese el paquete o clase de desarrollo. Luego, haga clic en el botón objeto local, que es lo mismo que escribir $TMP y presione grabar de nuevo.
    2. Escriba el siguiente código en el editor: WRITE: “Este es mi primer programa en ABAP”.
    3. Luego, presione el botón Activate, el cual compilará y activará el programa.
    • Todas las sentencias en ABAP terminan con un punto.
    • ABAP no distingue entre minúsculas y mayúsculas.
    • Una sentencia ABAP puede ocupar más de una línea.
    • Las cadenas de texto se escriben entre comillas simples.
    1. Una vez compilado y sin errores, ejecútelo con el botón correspondiente o presione la tecla F8.

    Reportes

    Siguen un modelo de programación relativamente simple, donde el usuario introduce una serie de parámetros y el programa los usa para producir reportes en forma de lista interactiva. El término reportes puede llevar a un equívoco, puesto que los reportes pueden ser diseñados para modificar datos. La razón por la que estos programas se llaman reportes es la “lista-orientada” que producen como salida.

    Module pool

    Definen patrones más complejos de interacción con el usuario a través de una colección de pantallas. El término pantalla se refiere a la imagen física que los usuarios pueden ver. Cada pantalla tiene un flujo lógico, el cual se refiere al código implícito de ABAP invocado por esta. Así mismo, cada flujo se divide en PBO (Process Before Output) y PAI (Process after Input). En la documentación de SAP el término dynpro (programa dinámico) se refiere a la combinación entre las pantallas y el flujo lógico.

    Include module

    Incluye el tiempo de generación en la unidad de llamada. Esto es usado en ocasiones para subdividir programas muy largos.

    Subroutine pools

    Contienen subrutinas ABAP (bloques de código cerrados por FORM/ENDFORM e invocados por PERFORM).

    Functions groups

    Son librerías de módulos de funciones (encerradas entre FUNCTION/ENDFUNCTION e invocadas por CALL FUNCTION).

    Object clases e interfaces

    Son similares a las clases e interfaces de Java. Las primeras definen una serie de métodos y atributos, y las segundas contienen la definición de los métodos. Para cada una de las clases implementadas se debe ofrecer el código de la misma.

    Type pools

    Definen una colección de datos, tipos y constantes.
